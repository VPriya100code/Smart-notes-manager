# 1. How to Run the Project

## Setup Steps

### 1️⃣ Create Virtual Environment

#### Windows

```bash
python -m venv venv

```

#### Mac/Linux

```bash
python3 -m venv venv

```

---

### 2️⃣ Install Dependencies

```bash
pip install -r requirements.txt
```

---

```bash

```

---

### 4️⃣ Open Browser

```bash
http://127.0.0.1:5000
```

The application will start locally and all notes will persist using SQLite database storage.

# 2. Stack Choice

I chose Flask with SQLite because I wanted a lightweight and simple stack that could be developed quickly while still supporting persistent storage.

Flask felt like the best choice for this assessment because:
- It is minimal and flexible
- Easy to structure for CRUD applications
- Works well with SQLite
- Requires very little setup

SQLite was chosen because:
- It stores data persistently in a local database file
- No separate database server is required
- Makes the project portable and easy to run on any machine

I avoided using heavier frameworks like Django because the project requirements were relatively small, and using a larger framework would have added unnecessary complexity for a mini-app.

# 3. One Real Edge Case

One edge case I handled was preventing users from submitting empty notes.

This validation is handled inside:

```bash
app.py
```

Before inserting a note into the database, the application checks whether the title or content fields are empty after trimming spaces.

Without this validation:
- Empty notes could be stored
- The UI would become cluttered
- The database would contain invalid entries

I also added delete confirmation prompts to help prevent accidental deletion of notes.

# 5. Honest Gap

While the application is fully functional, one area that could still be improved is note organization.

Currently, notes can only be marked as important, but they cannot be grouped into categories or folders.

With another day, I would implement:
- Categories and tags
- Drag-and-drop note arrangement
- User authentication
- Cloud synchronization
- Rich text editing support

I would also improve backend structure further by separating routes, database logic, and utility functions into different modules for better scalability and maintainability.
